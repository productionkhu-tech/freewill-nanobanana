# NanoBanana — 맥(macOS) 실행 가이드

> Windows에서는 `NanoBanana.exe`로 실행하지만, **맥에서는 소스에서 직접 실행하는 로컬 웹앱**으로 씁니다.
> (앱 본체가 원래 Flask 웹앱이라, 맥에서는 그걸 그대로 띄우고 **기본 브라우저**로 봅니다. Seedance 맥 방식과 동일.)

---

## 0. 준비물 (한 번만)

1. **Python 3** 설치 — https://www.python.org/downloads/macos/
   - 설치 후 터미널에서 확인: `python3 --version` (3.10 이상 권장)
   - ⚠️ python.org 설치본을 권장합니다(파일 선택창에 필요한 tkinter 포함).
2. **API 키** — 아래 중 가진 것만 있으면 됩니다 (셋 다 없어도 앱은 켜집니다).

---

## 1. 설치 (한 번만)

```bash
# 1) 앱 폴더 받기 (둘 중 하나)
git clone https://github.com/productionkhu-tech/freewill-nanobanana.git
cd freewill-nanobanana
#  또는 GitHub에서 Code → Download ZIP → 압축 풀고 그 폴더로 cd

# 2) 의존성 설치
pip3 install -r requirements_mac.txt

# 3) 실행 권한 부여 (더블클릭용)
chmod +x NanoBanana.command
```

### 키 등록
1. `keys.env.example` 파일을 복사해서 이름을 **`keys.env`** 로 바꿉니다.
2. 텍스트 편집기로 열어 가진 키만 채웁니다 (안 가진 줄은 비워둠).
3. **Vertex**를 쓴다면: 서비스계정 JSON 파일을 이 폴더에 두고, 파일명을 `GOOGLE_APPLICATION_CREDENTIALS=` 뒤에 적습니다 (예: `vertex-key.json`). 프로젝트 ID도 채웁니다.
4. 저장.

> `keys.env`와 키 JSON은 **본인 전용**입니다. 이미 `.gitignore`에 등록돼 있어 깃에 안 올라갑니다. 공유 금지.

---

## 2. 실행 (매번)

- **`NanoBanana.command` 더블클릭** → 잠시 후 기본 브라우저에 앱이 열립니다 (`http://127.0.0.1:5656`).
- 터미널 창은 **켜둔 채로** 사용하세요. 그 창을 닫으면 앱이 종료됩니다.
- 또는 터미널에서 직접: `python3 server_mac.py`

> ⚠️ 처음 더블클릭하면 맥이 "확인되지 않은 개발자" 경고를 낼 수 있어요.
> → `NanoBanana.command`를 **우클릭 → 열기**를 한 번만 해주면 이후엔 그냥 더블클릭됩니다.

---

## 3. 업데이트

**git clone으로 설치했다면, 실행할 때마다 자동으로 최신 버전을 받아옵니다.**
`NanoBanana.command`를 더블클릭하면 시작 전에 `git pull`이 돌아 항상 최신으로 켜집니다.

- 오프라인이거나 **ZIP으로 받았으면** 자동 업데이트를 건너뛰고 기존 버전으로 실행돼요.
- 수동으로 받고 싶으면: `cd freewill-nanobanana && git pull`
- 자동 업데이트를 끄려면: `keys.env`에서 `NANOBANANA_AUTO_UPDATE=0`
- 업데이트 후 가끔 새 패키지가 필요할 수 있어요 → `pip3 install -r requirements_mac.txt`

> 같은 코드(`main` 브랜치)를 Windows·맥이 공유합니다. 개발자가 한 번 push하면
> **Windows는 EXE 자동 교체로, 맥은 다음 실행 때 git pull로** 같은 버전을 받습니다.

---

## 4. 키 종류

| 환경변수 (keys.env) | 무엇 | 발급처 |
|---|---|---|
| `GOOGLE_APPLICATION_CREDENTIALS` + `NANOBANANA_PROJECT_ID` | Gemini (Vertex AI) | Google Cloud 서비스계정 JSON |
| `NANOBANANA_STUDIO_KEY` | Gemini (AI Studio, 가장 간단) | https://aistudio.google.com/apikey |
| `OPENAI_API_KEY` | gpt-image-2 모델 | https://platform.openai.com/api-keys |

세 가지를 다 넣을 필요는 없습니다. 가진 키에 해당하는 모델만 활성화됩니다.

## 5. 맥에서 되는 기능 / 차이

### ✅ 맥에서 그대로 되는 것 (거의 전부)
- **이미지 생성** — Gemini(Vertex·AI Studio) 3종 + OpenAI **gpt-image-2**
- 프로바이더 **자동 폴백**, 레이트리밋
- **레퍼런스 이미지** 슬롯 + `[Image N]` 멘션 + **드래그앤드롭 업로드**
- **종횡비 / 해상도 / Custom / Auto** 크기 (최근 작업 전부 포함)
- **갤러리** — 렌더·즐겨찾기·선택·삭제·마퀴
- **프롬프트 섹션 / 히스토리**
- **프로젝트 저장·로드** (JSON)
- **큐/배치 생성**, 진행률 바, **Stop**
- **PNG 메타데이터**, 파일 네이밍, 자동저장(15초)
- **이미지 뷰어** — 네이티브 창 대신 **인페이지 모달**로 (자동 폴백)
- **클립보드 이미지 붙여넣기** — 맥 경로 이미 구현돼 있음

### ⚠️ 차이 / 제한
| 항목 | Windows | 맥 |
|---|---|---|
| 자동 업데이트 | EXE 자동 교체 | 실행 시 자동 `git pull` (clone 설치 시) |
| 별도 OS 창(뷰어/프롬프트 팝업) | 네이티브 창 | 인페이지 모달/브라우저 탭 |
| "폴더에서 보기"(파인더 열기) | 됨 | 현재 미지원 (선택 작업으로 추가 가능) |
| 출력 폴더 **선택 다이얼로그** | 됨 | 맥에선 불안정할 수 있음 → **경로 직접 입력** 권장 |
| 단일 인스턴스·트레이·앱 아이콘 | 됨 | 브라우저라 해당 없음 |
| 실행 방식 | EXE 더블클릭 | Python 설치 + `NanoBanana.command` |

> 핵심 정리: **이미지 생성·레퍼런스·갤러리·프로젝트·크기 설정 같은 "앱의 본체 기능"은 100% 동일하게 작동**합니다. 차이는 대부분 Windows 네이티브 껍데기(자동업데이트·별도 창·탐색기 연동) 부분뿐입니다.

## 6. 문제 해결

| 증상 | 해결 |
|---|---|
| 브라우저가 자동으로 안 뜸 | 직접 주소창에 `http://127.0.0.1:5656` 입력 |
| "Address already in use" / 포트 사용 중 | 이미 실행 중인 것 — 기존 탭 사용, 또는 켜둔 터미널 창을 닫고 다시 실행 |
| `pip3: command not found` | Python 3 재설치(python.org 설치본 권장) |
| `Missing Python packages` 메시지 | `pip3 install -r requirements_mac.txt` 실행 |
| 키를 넣었는데 "연결 안 됨" | `keys.env`가 앱 폴더에 있는지, 철자/항목명 맞는지 확인 후 저장하고 재실행 |
| 출력 폴더 선택창이 안 뜸 | 출력 경로를 칸에 직접 입력 (또는 `brew install python-tk` 후 재시도) |
| Vertex가 연결 안 됨 | JSON 파일이 폴더에 있고 `GOOGLE_APPLICATION_CREDENTIALS` 파일명 + `NANOBANANA_PROJECT_ID`가 맞는지 확인 |

문제가 계속되면 터미널 창에 찍힌 메시지를 그대로 보내주세요 — 어디서 막혔는지 바로 진단됩니다.
